# 《莫须有之罪》网页设计架构文档

## 一、作品概述

**作品名称：** 莫须有之罪 —— AI 短剧行业困局与人的救赎

**核心命题：** AI 真的是短剧行业粗制滥造的罪魁祸首吗？—— 技术无罪，关键在于使用技术的人。

**叙事模型：** 剧本杀翻页对话式 + 梦中推演。用户以第一人称代入主角林野，在"梦境"中担任 AI 短剧公司老板，通过三封观众投诉信驱动的三轮调查辩论，逐步揭示真相：真正"有罪"的不是 AI，而是创作者的内容观念、审美判断与价值选择。

### 2.1 第一幕：入梦之前（场景 0-2）

| 场景 | 名称 | 功能 | 关键元素 |
|------|------|------|----------|
| 0 | **开机** | 黑场→快门闪白→封面浮现，电影开场仪式感 | 相机快门音效（Web Audio API 合成）、两帧闪白过渡 |
| 1 | **人物** | 翻转卡片式角色档案，"你将扮演：林野" | 3D 翻转动画、档案卷宗红色印章 |

### 2.2 第二幕：梦中推演（场景 3-9）—— 三轮脱罪

梦境以**眨眼动画**进入，林野"穿越"为 AI 短剧公司老板，助手贵哥陪同调查三封观众投诉信：

#### 第一轮脱罪：「情绪」—— 场景 4 → 脱罪 1（场景 5）

| 投诉 | 调查手段 | 核心论据 | 结论 |
|------|----------|----------|------|
| 观众反感：AI 批量生产低质情绪内容 | 爽点设置分析 + AI 制作流程 + 行业访谈（陈楸帆、汪家城、赵晖） | AI 只能学习情绪规律，人类把控内容方向 | **"技术负责放大情绪，人负责理解情绪"** |

#### 第二轮脱罪：「兴趣」—— 场景 6 → 脱罪 2（场景 7）

| 投诉 | 调查手段 | 核心论据 | 结论 |
|------|----------|----------|------|
| 观众反映：抄袭建模、雷同剧情、无原创方向 | 行业题材四象限气泡图 + 金字塔流量模型 + 男女频取向分析 + 访谈（丁宽、缪轶、赵晖） | 同质化源于市场压力下的安全选择，非 AI 之过 | **"技术负责降低创作门槛，人负责突破创作边界"** |

#### 第三轮脱罪：「审美」—— 场景 8 → 脱罪 3（场景 9）

| 投诉 | 调查手段 | 核心论据 | 结论 |
|------|----------|----------|------|
| 观众质疑：AI 方便粗制滥造，拉低行业下限 | 5 部优秀 AI 短剧案例 + AI 劝退密码解析 + 寄语（赵姐、赵晖） | AI 同样能产出高品质作品，审美判断力才是分水岭 | **"技术负责提升下限，人负责决定上限"** |

### 2.3 第三幕：梦醒之后（场景 10-12）

| 场景 | 名称 | 功能 | 关键元素 |
|------|------|------|----------|
| 10 | **梦醒** | 视频揭晓"原来是梦"，快门声对称回应开机 | 视频播放 + 眨眼过渡 + 相机快门音效 |
| 11 | **结语** | 直陈主旨，三段递进式反思 | "AI短剧的未来，不在算法里，而在人心里" |
| 12 | **谢幕** | 制作团队、参考文献、机构标识 | 8 图轮播、5 篇学术文献 |


## 三、人物体系

### 3.1 主角：林野

- **身份：** 虚构人物，原型来自实地采访的传统短剧/影视编剧群体
- **人物内核：** 与《月亮与六便士》主人公高度契合 —— 困在艺术理想与金钱现实的两难中
- **设计手法：** 用户以第一人称代入，直接体验林野的困惑、调查与觉醒
- **视觉呈现：** 暗调人物剪影、逆光肖像

### 3.2 配角：贵哥

- **身份：** AI 短剧公司老板助理，梦境中的"引导者"
- **功能：** 辩论对手 + 信息提供者，推动三轮脱罪论证
- **设计手法：** 代表 AI 行业理性声音，用数据与案例回应林野的质疑

### 3.3 访谈人物群像

| 人物 | 身份 | 出现场景 | 核心观点 |
|------|------|----------|----------|
| 陈楸帆 | 科幻作家、《神·笔》总编剧 | 情绪的垃圾场 | "AI 完成 80% 工业化工作，最后 20% 灵魂由人完成" |
| 汪家城 | 九州文化编剧负责人 | 情绪的垃圾场 | "审美和判断，才是最值钱的资产" |
| 赵晖 | 中国传媒大学教授 | 情绪/兴趣/审美 | "人机协作进行视听创作的时代到来了" |
| 丁宽 | AI 导演 | 兴趣的流水线 | "用世界通用类型片语言讲好中国故事" |
| 缪轶 | 内容创作者 | 兴趣的流水线 | "工业化制作管线 + 编剧团队 + 版权合规" |

## 四、视觉设计体系

### 4.1 色彩方案

**设计来源：** 提取《月亮与六便士》经典封面配色

```
┌─────────────────────────────────────────────────────────────────┐
│                        色彩三幕演变                              │
│                                                                 │
│   第一幕（紫）          第二幕（琥珀）        第三幕（灰）        │
│   #1E112F ─────────→   #3D2616 ─────────→   #1A1A1A            │
│   暗夜 · 神秘 · 入梦    烛光 · 调查 · 揭露    清醒 · 结论 · 回归  │
│                                                                 │
│   主色 #D4A843（暗金）贯穿全场，对应理想之光                       │
│   辅色 #E8B84A（琥珀金）点缀高亮                                 │
│   底色 #0D0A12（深紫黑）营造悬疑氛围                              │
│   亮色 #F5E6C8（奶油白）用于正文文字                              │
└─────────────────────────────────────────────────────────────────┘
```

**冷暖撞色逻辑：** 深紫（冷）vs 明黄（暖）= 理想 vs 现实的对立冲突

### 4.2 视觉调性

**参考来源：** 英伦悬疑剧《喜鹊谋杀案》视觉风格

| 设计元素 | 应用方式 |
|----------|----------|
| **高对比度几何块面** | 场景边框、分隔线、卡片阴影 |
| **人物剪影** | 林野与贵哥的圆顶高挑肖像（`rounded-t-[160px]`） |
| **胶片电影元素** | 两侧胶片齿孔装饰、letterbox 遮幅、grain 噪点、dust 划痕 |
| **警戒线符号** | 投诉信弹窗红色渐变顶条、"紧急"标签 |
| **场记板符号** | 嵌入式图表 HTML 中反复出现的拍板 SVG |
| **取景框四角** | 固定四角 L 形标记（`pointer-events-none fixed inset-10`） |
| **满月意象** | 月光脉冲动画（`moonPulse`），呼应《月亮与六便士》 |

## 五、交互设计

### 5.1 核心交互模式

| 交互 | 实现 | 说明 |
|------|------|------|
| **主翻页** | CSS snap-scroll（`scroll-snap-type: y mandatory`） | 13 个全屏场景纵向吸附滑动 |
| **场景内翻页** | 点击区域 / 左右箭头 | 左侧图片 + 右侧文字面板内的子页面 |
| **对话推进** | 点击图片区域 / 箭头按钮 | 多轮对话逐步展开 |
| **数据弹窗** | 按钮触发 → iframe/portal 弹窗 | 行业报告、图表、访谈 |
| **场景跳转** | `CustomEvent("scrollToSection")` | 按钮触发跨场景导航 |
| **键盘导航** | ↑↓ 方向键 | 上下切换场景 |
| **侧边导航** | 右侧圆点点击 | 13 个场景直接跳转 |

### 7.3 跨场景通信

场景间不通过 props 传递，而是使用浏览器 CustomEvent：

```typescript
// 场景内发送跳转信号
window.dispatchEvent(new CustomEvent("scrollToSection", {
  detail: { index: 4, section: "studio", triggerTransition: true }
}));

// Root.tsx 统一监听并执行滚动
window.addEventListener("scrollToSection", (e) => scrollToSection(e.detail.index));
```

### 7.4 音频架构

```
src/audio/index.ts 统一管理
├── prologue: [p1, p2, null, p3]      ← 序幕
├── actoneDialogues: [d1-d4]          ← 入梦对话
├── actoneBg: [bg1, bg2]              ← 入梦背景音乐（链式播放）
├── studio: [s1-s4, null]             ← 情绪的垃圾场
├── extra: [e1-e3, e8, e4, null, e7, null] ← 兴趣的流水线
├── extra2: [e1-e3, null, e5, null, e7-e9] ← 审美的失乐园
└── dreamReveal                       ← 梦醒
```

每个对话场景采用 `Map<stepIndex, HTMLAudioElement>` 模式，`useEffect` 监听 `[isActive, step]` 实现"翻到该句自动播放、划走自动停止"。

### 7.5 图片预热机制

所有场景组件在挂载时执行预热函数，使用原生 `Image.decode()` 异步解码：

```typescript
function warmImages() {
  imageSources.forEach((src) => {
    const img = new Image();
    img.src = src;
    img.decoding = "async";
    img.decode?.().catch(() => undefined);
  });
}
```

CDN 脚本（ECharts、Tailwind CSS）通过 `<link rel="preload" as="script">` 提前缓存。
